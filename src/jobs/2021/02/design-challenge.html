Challenge Description Instruktion

Learnpoint is organized in courses and each course has a document tool. Users can add, search, and view documents. Learnpoint innehåller kurser och varje kurs har en dokumentverktyg. Användare kan lägga till, söka, och titta på dokument.

The documents are sorted alphabetically, but there is no other structure. If there are lots of documents, it is difficult for the user to get a clear overview of all the documents. Dokumenten är sorterade i bokstavsordning, men i övrigt finns ingen struktur. Om det finns många dokument är det svårt för användare att få en god överblick av alla dokument.

The problem is not about finding a specific document. If the user knows what document she is looking for, she can use the search feature. The problem is about getting a clear overview if there are lots of documents. Problemet handlar inte om att hitta ett visst dokument. Om användaren vet vilket dokument hon letar efter, så kan hon använda sökfunktionen. Problemet handlar om att få en god överblick när det finns väldigt många dokument.

The challenge has two tasks: Provet består av två uppgifter:

  1. Find and discuss different solutions to the overview problem. Identifiera och diskutera olika lösningar på problemet med översikt.
  2. Implement one of your solutions. Implementera en av dina lösningar.

How to approach the challange Tips för att lösa uppgiften

This design challenge is primarily about showcasing your work processess and the way you think about design problems. When your work is evaluated, the first task will be more important than the second. Make sure to spend enough time on the first task before moving on to the next. Designprovet skall framför allt visa hur du arbetar och resonerar kring ett designproblem. Ditt arbete kommer bedömas med tonvikt på den första uppgiften. Se till att lägga tillräckligt med tid på den första uppgiften innan du går vidare till nästa.

The second task should not result in a complete implementation that's ready for deployment. But it needs to be finished enough to demonstrate your chosen solution. If the solution requires a lot of work, you can limit the implementation to just a part of the solution. To get started, you can use the source code from this example. Den andra uppgiften ska inte resultera i en fullständig implementation som är färdig att leverera. Däremot behöver den vara tillräckligt genomarbetad för att demonstrera din valda lösning. Om lösningen är omfattande, kan du begränsa implementationen till en del av lösningen. För att komma igång kan du använda källkoden till detta exempel.

Handing in your work Inlämning

The first task should be documented in a PDF and the second task should be coded in a HTML file. The files should be zipped before you hand them in. Den första uppgiften skall dokumenteras i en PDF och den andra skall kodas i en HTML-fil. Zippa filerna innan du lämnar in dom.

(Alternatively, you can write a blog post for the first task and/or publish the second task on the web. If you choose this option, you should hand in links instead of zipped files.) (Alternativt kan du skriva ett blogginlägg för den första uppgiften och/eller publicera den andra uppgiften på webben. Om du väljer detta alternativ så lämnar du in länkar istället för zippade filer.)
